# Delegation Rules

## 基本方針

デフォルトは常に委譲。メインループが自分で直接手を動かしてよいのは次の2つに限る:

- (a) ユーザーとの対話・確認・質問。単一ファイル・単一情報源を読めば即答できる事実確認を含む。複数の情報源を横断する、または結論を合成する必要がある行為は「調査」であり (a) に含めない。
- (b) サブエージェントが返した出力のレビュー・要約・編纂・ファイルへの反映。新規の内容生成・構成判断を伴わない転記・整形はここに含む。新たな文章構成や解釈が必要な場合はロール別ルーティング表に従う。

上記2つに該当しない行為(調査・実装・分析・デバッグ・計算・執筆・検証など)を行おうとした瞬間、それがどれだけ簡単に見えても、領分や難易度を検討せずに該当エージェントを spawn する。「これは自分でもできる」という自己評価は判定に使わない。着手済みの作業を継続する場合も、行動の節目ごとにこの原則へ照合し直す。

(a)(b) への該当が判定できた時点で、以下のロール別ルーティング表の参照は不要である。表は (a)(b) 非該当の行為にのみ適用する。

## ロール別ルーティング

| 作業内容 | エージェント | モデル |
|---|---|---|
| コード実装・テスト作成・リファクタリング・CI/CD・インフラ構築・計算コスト見積もり・運用ダッシュボード | `engineer` | Opus |
| 仮説・メカニズムの検討/結果解釈・実験設計・統計分析・統計的可視化 | `analyst` | Opus |
| 一般的な調査・事実確認・文献サーベイ・要約・競合調査 | `researcher` | Sonnet |
| 文書執筆・編集(論文・提案書・技術文書・レポート・ノート) | `writer` | Sonnet |
| 上記に当てはまらない探索的な調査・コード検索 | `Explore` / `general-purpose` | メイン継承 |

具体的な役割が定義済みのタスク(project-team 系ではなく単発の実装/分析/調査/執筆)では、`general-purpose` より上記の役割別エージェントを優先する。

## 報告前レビュー

サブエージェントの出力をそのまま転記しない。内容を検証し、必要なら自分の言葉で整えてから報告する。

`writer` は Write/Edit ツールを持たない。成果物は本文としてテキストで返ってくるので、メインループが内容を検証したうえで、実際のファイルへの保存を行う。
