# Phase 6: Output (Paper / Product)

Task generation template for the deliverable creation phase.
SKILL.md reads this file when entering Phase 6 and generates tasks dynamically
based on the Phase 5 analysis report.

## Input

- Phase 5 deliverable: **Analysis report** (statistical summary, figures, hypothesis verdict = supported)
- Phase 1 deliverable: **Survey report** (for related work section / competitive positioning)

## Task Generation Rules

### Academic Mode

1. **Paper structure planning** (1 task)
   - Agent: none (human task — lead decides the story arc)
   - Dependencies: none
   - Instruction: Lead outlines the paper structure: key claim, narrative flow, which results to highlight. SKILL.md prompts the lead to provide a 3-5 bullet outline

2. **Introduction and Related Work** (1 task)
   - Agent: Research (execute)
   - Dependencies: structure plan from lead
   - Instruction: Draft Introduction (motivation, contribution list) and Related Work (from Phase 1 survey, positioned against the gap). Use `/reference-verify` to confirm all citations exist

3. **Method section** (1 task)
   - Agent: Research (execute)
   - Dependencies: structure plan from lead
   - Instruction: Draft the Method section describing the proposed approach. Cross-reference the Phase 3 implementation to ensure accuracy

4. **Experiments and Results section** (1 task)
   - Agent: Research (execute)
   - Dependencies: structure plan from lead
   - Instruction: Draft Experiments (setup, baselines, metrics) and Results (statistical findings, figures from Phase 5). Integrate figures and tables generated by the Analyst

5. **Discussion and Conclusion** (1 task)
   - Agent: Research (execute)
   - Dependencies: categories 2, 3, 4 complete
   - Instruction: Draft Discussion (interpretation, limitations, future work) and Conclusion. Ensure claims do not overreach the statistical evidence

6. **Reproducibility package** (1 task)
   - Agent: Engineer (execute)
   - Dependencies: categories 2-5 complete
   - Instruction: Prepare code for release: clean up repository, add README with reproduction instructions, verify Docker build, create requirements.txt / pyproject.toml

7. **Claim-evidence review** (1 task)
   - Agent: Analyst (critique)
   - Dependencies: categories 2-5 complete
   - Instruction: Review the paper draft for claim-evidence alignment. Verify every quantitative claim matches the Phase 5 statistical summary. Flag unsupported or overstated claims

### Startup Mode

1. **Deliverable planning** (1 task)
   - Agent: none (human task — lead decides what to produce)
   - Dependencies: none
   - Instruction: Lead specifies deliverables: technical documentation, investor pitch deck, deployment plan, or combination. SKILL.md prompts the lead to select

2. **Technical documentation** (1 task, if selected)
   - Agent: Research (execute)
   - Dependencies: deliverable plan from lead
   - Instruction: Write technical documentation: system architecture, API reference, deployment guide, performance benchmarks

3. **Investor / stakeholder materials** (1 task, if selected)
   - Agent: Research (execute)
   - Dependencies: deliverable plan from lead
   - Instruction: Draft investor deck or stakeholder report: problem, solution, market, traction, competitive advantage. Ground claims in Phase 5 data

4. **Production code preparation** (1 task)
   - Agent: Engineer (execute)
   - Dependencies: deliverable plan from lead
   - Instruction: Production-ready code: API packaging, edge optimization, monitoring setup, CI/CD pipeline, deployment scripts

5. **Claims review** (1 task)
   - Agent: Analyst (critique)
   - Dependencies: categories 2-4 complete (whichever were generated)
   - Instruction: Review all deliverables for statistical accuracy. Verify performance claims match experiment data

## Task Granularity Guidelines

- 1 task = 1 agent invocation
- Paper sections are split by standard structure (Intro+Related, Method, Experiments+Results, Discussion+Conclusion) — not by paragraph
- If a section exceeds ~3000 words, consider splitting (rare for conference papers)
- The structure/deliverable planning step is mandatory human input

## Completion Condition

Output draft (paper or product) complete.
See `references/phase_transitions.md` for transition rules.
